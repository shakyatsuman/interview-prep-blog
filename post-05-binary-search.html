Binary Search

Eliminate half the search space on every step. The classic O(log n) pattern — and the powerful search space reduction generalization that goes far beyond sorted arrays.

⏱ 5 min read 🔍 Fundamentals 🔑 Java 8 📈 Intermediate → Advanced

A linear scan through n elements costs O(n). Binary search reduces this to O(log n) by exploiting one property: the search space is monotonic — every step can definitively eliminate half the remaining candidates. On a sorted array of one million elements, binary search finds any value in at most 20 comparisons. This post covers the classic form and the more powerful search space reduction generalization that applies to problems far beyond simple sorted arrays.

The Core Idea

Binary search maintains a search space bounded by lo and hi. On each iteration it computes mid = lo + (hi - lo) / 2 and uses the value at mid to decide which half to discard. The key requirement: the array must be sorted, or the condition being searched must be monotonic — if the condition is false at index i, it must be false for all indices below i (or above, depending on direction).

Fig 1. Binary search on a sorted array. Each step halves the search space — O(log n) total comparisons.

The mid formula

Always use mid = lo + (hi - lo) / 2, not (lo + hi) / 2. The latter overflows when lo and hi are large integers. In Java 8 this is a real bug — not a theoretical concern.

Two Flavors of Binary Search

Classic — Find Exact Value

Search a sorted array for a specific target. The loop runs while lo <= hi. Return the index when found, or -1 if the target is absent.

Search Space Reduction — Find Boundary

The more powerful form. Instead of looking for an exact value, search for the leftmost position where a condition becomes true. The loop runs while lo < hi. Use when the problem asks for a minimum value satisfying a constraint, or the first index where something changes.

Example 1 — Binary Search

LeetCode #704 · Easy

Given a sorted array of distinct integers and a target, return the index of the target or -1 if it does not exist.

Example

nums = [-1, 0, 3, 5, 9, 12], target = 9 → 4
nums = [-1, 0, 3, 5, 9, 12], target = 2 → -1

Fig 2. LC 704 trace. Two steps to find target=9; a linear scan would check 5 elements.

Java 8public int search(int[] nums, int target) {
    int lo = 0;
    int hi = nums.length - 1;

    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2; // avoids integer overflow

        if (nums[mid] == target) {
            return mid;
        } else if (nums[mid] < target) {
            lo = mid + 1;  // target is in right half
        } else {
            hi = mid - 1;  // target is in left half
        }
    }

    return -1; // target not found
}
// Time: O(log n)  |  Space: O(1)

Example 2 — Koko Eating Bananas

LeetCode #875 · Medium · Search space reduction

Koko has piles of bananas and h hours to eat them all. Each hour she eats at most k bananas from one pile. Return the minimum eating speed k such that she can finish all piles within h hours.

Example

piles = [3, 6, 7, 11], h = 8 → 4
At speed 4: ⌈3/4⌉+⌈6/4⌉+⌈7/4⌉+⌈11/4⌉ = 1+2+2+3 = 8 hours ✓

There is no sorted array here. The insight: feasibility is monotonic in k — if speed k works, any speed k' > k also works. This means valid speeds form a contiguous suffix of [1..max(piles)]. Binary search finds the left boundary: the smallest k where canFinish(piles, k, h) is true.

Fig 3. Koko search space. Feasibility is monotonic: all ✗ then all ✓. Binary search finds the leftmost ✓ in O(log m) steps.

Java 8public int minEatingSpeed(int[] piles, int h) {
    int lo = 1;
    int hi = 0;
    for (int p : piles) hi = Math.max(hi, p); // max pile = upper bound

    while (lo < hi) {                          // find leftmost valid k
        int mid = lo + (hi - lo) / 2;
        if (canFinish(piles, mid, h)) {
            hi = mid;        // mid works; try smaller
        } else {
            lo = mid + 1;   // mid too slow; need faster
        }
    }

    return lo; // lo == hi == minimum valid speed
}

private boolean canFinish(int[] piles, int k, int h) {
    int hours = 0;
    for (int p : piles) hours += (p + k - 1) / k; // ceiling division
    return hours <= h;
}
// Time: O(n log m)  where m = max(piles)  |  Space: O(1)
Pro Tip

The ceiling division (p + k - 1) / k is equivalent to Math.ceil((double)p / k) but avoids floating-point arithmetic entirely. Interviewers notice this.

Universal Template

The two canonical binary search skeletons:

Java 8 — Pattern B: Find Boundary// Find leftmost position where condition is true
int lo = minSearchSpace, hi = maxSearchSpace;
while (lo < hi) {
    int mid = lo + (hi - lo) / 2;
    if (condition(mid)) {
        hi = mid;       // mid satisfies; try smaller
    } else {
        lo = mid + 1;  // mid fails; discard left half
    }
}
return lo; // leftmost position where condition is true

When to Reach for Binary Search

  • The input array is sorted, or the problem says "find in sorted"
  • "Find the minimum/maximum value satisfying a constraint" — classic search space reduction
  • A feasibility check (canFinish, isValid) is monotonic: if true at x, true for all x' > x
  • O(n) feels too slow and the answer lies in a bounded range
  • The word "sorted" appears, or sorting first would not destroy useful information
