Sliding Window

A specialization of same-direction two pointers that maintains a dynamic subarray satisfying a constraint — turning O(n²) substring scans into O(n) single-pass solutions.

⏱ 5 min read 🔮 Fundamentals 🔑 Java 8 📈 Intermediate → Advanced
Prerequisites

This post builds directly on Post #2 (Two Pointers) — specifically the fast/slow same-direction pointer pattern. Sliding window is a direct extension of that idea applied to subarrays with constraints.

In Post #2 we saw that same-direction pointers can partition or deduplicate an array in a single pass. Sliding window extends this: the right pointer always advances, expanding the window, while the left pointer advances only when a constraint is violated, shrinking it back to validity. The result is an O(n) algorithm that would naively require an O(n²) nested scan.

The Core Idea

A window is a contiguous subarray defined by two indices, left and right. As right sweeps forward, each element enters the window exactly once. The left pointer advances to evict elements only when necessary. Because every element is added at most once and removed at most once, the total work across all iterations is O(n) — not O(n²).

Fig 1. The window [c, d, e] is bounded by left and right. Every element enters and exits the window at most once — O(n) total.

Two Flavors of Sliding Window

Fixed-Size Window

The window size is a constant k. Both pointers advance together one step at a time. On each step, subtract the element leaving the left edge and add the element entering the right edge. Use when the problem specifies a fixed subarray length.

Variable-Size Window

The window grows and shrinks based on a constraint. right always advances; left advances only when the window becomes invalid. Use when the problem asks for the longest or shortest subarray satisfying a condition.

Example 1 — Maximum Sum Subarray of Size k

Fixed-size window

Given an integer array and a number k, find the maximum sum of any contiguous subarray of size k.

Example

arr = [2, 1, 5, 1, 3, 2], k = 3 → 9 (subarray [5, 1, 3])

The key optimization: instead of recomputing the full window sum on each step, subtract the outgoing element and add the incoming element. Each step is O(1).

Fig 2. Four windows of size 3. Step 3 (green) yields the maximum sum of 9.

Java 8public int maxSumSubarray(int[] arr, int k) {
    int windowSum = 0;

    for (int i = 0; i < k; i++) {
        windowSum += arr[i];          // build initial window
    }

    int maxSum = windowSum;

    for (int i = 1; i <= arr.length - k; i++) {
        windowSum += arr[i + k - 1] - arr[i - 1]; // slide: add right, remove left
        maxSum = Math.max(maxSum, windowSum);
    }

    return maxSum;
}
// Time: O(n)  |  Space: O(1)

Example 2 — Longest Substring Without Repeating Characters

LeetCode #3 · Medium · Variable-size window

Given a string s, return the length of the longest substring containing all unique characters.

Example

s = "abcabcbb" → 3 (substring "abc")

The window expands by adding s.charAt(right) to a HashSet. When a duplicate is detected, the left pointer advances — evicting characters one by one — until the window is valid again.

Fig 3. Variable window through "abcabcbb". Dashed red = duplicate detected, shrink left. Green = new max length of 4 at Step 3.

Java 8public int lengthOfLongestSubstring(String s) {
    Set<Character> window = new HashSet<>();
    int left   = 0;
    int maxLen = 0;

    for (int right = 0; right < s.length(); right++) {
        // shrink from left until the duplicate is evicted
        while (window.contains(s.charAt(right))) {
            window.remove(s.charAt(left));
            left++;
        }

        window.add(s.charAt(right));
        maxLen = Math.max(maxLen, right - left + 1);
    }

    return maxLen;
}
// Time: O(n)  |  Space: O(min(n, |alphabet|))
Optimization

Replace the HashSet with a HashMap<Character, Integer> mapping each character to its last seen index. When a duplicate is found, jump left directly to lastSeen.get(c) + 1 rather than walking it forward one step at a time. Eliminates the inner while loop entirely.

Universal Template

Almost every variable sliding window problem fits this skeleton:

Java 8 — Variable Windowpublic int slidingWindow(int[] arr) {
    int left   = 0;
    int result = 0;
    // window state: HashMap, HashSet, int counter, etc.

    for (int right = 0; right < arr.length; right++) {
        // 1. Expand: add arr[right] to window state

        // 2. Shrink: move left while constraint is violated
        while (/* constraint violated */) {
            // remove arr[left] from window state
            left++;
        }

        // 3. Update result (window is now valid)
        result = Math.max(result, right - left + 1);
    }

    return result;
}

When to Reach for Sliding Window

Recognize these signals in a problem statement:

  • Words like "subarray", "substring", or "contiguous" paired with a max, min, or sum
  • A constraint like "at most k distinct elements" or "no repeating characters"
  • The input is a linear sequence — array or string
  • A fixed window size k is given explicitly (fixed variant)
  • You need the longest or shortest valid subarray (variable variant)
