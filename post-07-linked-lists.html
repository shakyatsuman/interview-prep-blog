Linked Lists

Pointer manipulation without indices. Master the dummy node, in-place reversal, and the fast/slow pointer — three building blocks that underpin trees, graphs, and beyond.

⏱ 5 min read 🔗 Data Structures 🔑 Java 8 📈 Intermediate → Advanced

A linked list is a sequence of nodes, each holding a value and a pointer to the next node. Unlike arrays, nodes are not contiguous in memory — there is no index arithmetic, only pointer traversal. This changes everything: random access is O(n), but insertion and deletion at a known position are O(1). The interview challenge is not the data structure itself but reasoning carefully about pointer manipulation without losing references to nodes you still need.

Prerequisites

Post #2 (Two Pointers) — the fast/slow same-direction pattern. The linked list version is the same idea applied to a non-indexed structure.

Three Core Patterns

The Dummy Node

A dummy (sentinel) head node is a placeholder prepended to the list before any operation. It eliminates the special case of modifying the real head, allowing every node — including the first — to be treated uniformly. The result is returned as dummy.next.

Fig 1. The dummy node (D) gives node 1 a predecessor, eliminating the head special case. Return dummy.next as the result.

In-Place Reversal

Reversing a linked list in-place requires three pointers: prev, curr, and next. On each step: save curr.next before overwriting it, redirect the pointer backward, then advance all three. No extra space needed.

Fast / Slow Pointers

Two pointers starting at the head, advancing at different rates — fast moves two steps per iteration, slow moves one. When fast reaches the end, slow is at the midpoint. If the list has a cycle, fast and slow will eventually meet inside it. This is the same pattern from Post #2, now applied to a non-indexed structure.

Example 1 — Reverse Linked List

LeetCode #206 · Easy · In-place reversal

Given the head of a singly linked list, reverse the list and return the new head.

Example

1 → 2 → 3 → 4 → 5 → null → 5 → 4 → 3 → 2 → 1 → null

Three pointers carry the state: prev (the new next for curr), curr (the node being redirected), and next (saved before the pointer is overwritten). Each iteration is O(1); the full pass is O(n).

Fig 2. Three-pointer reversal trace on [1→2→3]. Each step: save next, redirect curr.next to prev, advance both. After n steps, prev is the new head.

Java 8public ListNode reverseList(ListNode head) {
    ListNode prev = null;
    ListNode curr = head;

    while (curr != null) {
        ListNode next = curr.next;  // save next before overwriting
        curr.next = prev;           // redirect pointer backward
        prev = curr;                // advance prev
        curr = next;                // advance curr
    }

    return prev;  // prev is the new head
}
// Time: O(n)  |  Space: O(1)
Common Mistake

Forgetting to save curr.next before overwriting it. Once curr.next = prev executes, the original next node is unreachable unless you stored it first. Always: save, redirect, advance.

Example 2 — Linked List Cycle Detection

LeetCode #141 · Easy · Fast/slow pointers

Given the head of a linked list, return true if the list contains a cycle, false otherwise. A cycle exists when a node's next pointer points back to a previous node in the list.

Example

3 → 2 → 0 → -4 → (back to 2) → true
1 → 2 → null → false

Floyd's cycle detection algorithm: advance fast by two nodes and slow by one per iteration. If there is no cycle, fast reaches null first. If there is a cycle, fast and slow must eventually meet inside it — the faster pointer laps the slower one within the cycle.

Fig 3. Floyd's algorithm on 3→2→0→-4→(2). Fast laps slow inside the cycle within 3 steps. No cycle: fast reaches null first.

Java 8public boolean hasCycle(ListNode head) {
    ListNode slow = head;
    ListNode fast = head;

    while (fast != null && fast.next != null) {
        slow = slow.next;        // advance one step
        fast = fast.next.next;   // advance two steps

        if (slow == fast) return true;  // cycle detected
    }

    return false;  // fast reached end: no cycle
}
// Time: O(n)  |  Space: O(1)
Why They Must Meet

In a cycle of length L, fast gains one node on slow per iteration. After at most L steps inside the cycle, fast has lapped slow and they occupy the same node. The gap between them decreases by 1 each iteration until it reaches 0.

When to Reach for These Patterns

  • Modifying the head node or inserting at the front — use a dummy node
  • "Reverse" or "rotate" a list or a sublist — in-place reversal
  • "Find the middle" or "does a cycle exist" — fast/slow pointers
  • "Remove nth node from end" — fast/slow with an n-step head start on fast
  • "Check if palindrome" — find middle with fast/slow, reverse second half, compare
