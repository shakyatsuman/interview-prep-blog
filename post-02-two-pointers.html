Two Pointers

Eliminate the inner loop. Two indices moving through an array in a single pass unlock O(n) solutions that sorting alone cannot reach.

⏱ 5 min read 🌿 Fundamentals 🔑 Java 8 📈 Intermediate → Advanced

The two-pointer technique places two indices into an array and moves them — toward each other, or in the same direction at different speeds — to solve problems that would otherwise require a nested loop. The prerequisite insight from Post #1 applies directly: when you know something about the structure of your data, you can skip work rather than checking every pair. Here, sorting provides that structure.

The Core Idea

A brute-force search over all pairs costs O(n²). Two pointers reduce this to O(n) by exploiting a monotonic relationship: on a sorted array, if the current sum of two elements is too large, moving the right pointer left will decrease it; if too small, moving the left pointer right will increase it. Each pointer moves at most n steps total, giving O(n) work.

Fig 1. Opposite-end pointers on a sorted array. Each move eliminates one element from consideration — no element is visited more than once.

Two Flavors of Two Pointers

Opposite-End Pointers

Left starts at index 0, right starts at the last index. They converge toward the centre. Use when the array is sorted and you need to find a pair satisfying a condition — a sum, product, or distance constraint.

Same-Direction Pointers (Fast / Slow)

Both pointers start at the same end and advance at different rates. Use for partitioning, removing duplicates in-place, or detecting cycles in linked lists. The slow pointer marks the write position; the fast pointer scans ahead.

Fig 2. Fast/slow pointers removing duplicates in-place. Slow marks the write position; Fast scans ahead and copies unique values.

Example 1 — Two Sum II (Sorted Array)

LeetCode #167 · Difficulty: Medium

Given a 1-indexed sorted array of integers and a target, return the indices of the two numbers that add up to target. Exactly one solution exists; each element may be used at most once.

Example

numbers = [2, 7, 11, 15], target = 9 → [1, 2] because numbers[1] + numbers[2] = 2 + 7 = 9

Because the array is sorted, the sum of numbers[left] + numbers[right] changes predictably as each pointer moves. This is the monotonic property that makes two pointers work here.

Fig 3. Two Sum II trace. Three steps to find the answer; brute force would check all 6 pairs.

Java 8public int[] twoSum(int[] numbers, int target) {
    int left  = 0;
    int right = numbers.length - 1;

    while (left < right) {
        int sum = numbers[left] + numbers[right];

        if (sum == target) {
            return new int[] { left + 1, right + 1 }; // 1-indexed
        } else if (sum < target) {
            left++;   // sum too small — increase by moving left pointer right
        } else {
            right--;  // sum too large — decrease by moving right pointer left
        }
    }

    return new int[] {}; // unreachable per problem guarantee
}
// Time: O(n)  |  Space: O(1)

Example 2 — Remove Duplicates from Sorted Array

LeetCode #26 · Easy · Same-direction fast/slow pointers

Given an integer array sorted in non-decreasing order, remove duplicates in-place so each unique element appears only once. Return the count of unique elements. The relative order must be preserved and O(1) extra space must be used.

Example

nums = [1, 1, 2, 3, 3, 4] → returns 4, array modified to [1, 2, 3, 4, _, _]

The slow pointer marks the write position — the last confirmed unique element. The fast pointer scans ahead. When it finds an element different from nums[slow], a new unique element has been discovered: advance slow and overwrite. Because the array is sorted, all duplicates of any value are contiguous, so this single pass is sufficient.

Fig 4. Fast/slow same-direction pointers. Green = current write position (S). Dashed red = duplicate being skipped. Each unique value is written exactly once.

Java 8public int removeDuplicates(int[] nums) {
    if (nums.length == 0) return 0;

    int slow = 0;  // write position — last confirmed unique element

    for (int fast = 1; fast < nums.length; fast++) {
        if (nums[fast] != nums[slow]) {  // new unique element found
            slow++;
            nums[slow] = nums[fast];       // write it into the next position
        }
        // if equal: fast advances, slow stays — duplicate is skipped
    }

    return slow + 1;  // number of unique elements
}
// Time: O(n)  |  Space: O(1)
Key Insight

The array being sorted is what makes this work: all copies of any duplicate are contiguous, so a single left-to-right pass encounters each unique value exactly once. On an unsorted array you would need a HashSet — O(n) space. Sorting first costs O(n log n) but then enables this O(1)-space pass.

Universal Template

The two canonical forms of the two-pointer pattern:

Java 8 — Opposite-End// Pattern A: opposite-end pointers (sorted array, pair search)
int left  = 0;
int right = arr.length - 1;

while (left < right) {
    int val = compute(arr[left], arr[right]);
    if      (val == target) { /* found */ }
    else if (val < target)  { left++;  }  // need larger value
    else                    { right--; }  // need smaller value
}
Java 8 — Fast / Slow// Pattern B: same-direction fast/slow (partition, dedup)
int slow = 0; // write position

for (int fast = 1; fast < arr.length; fast++) {
    if (arr[fast] != arr[slow]) { // new unique element found
        slow++;
        arr[slow] = arr[fast];   // write it into position
    }
    // if equal: fast advances, slow stays — duplicate is skipped
}

When to choose Two Pointers over HashMap

Two pointers win when: (1) you need O(1) space, (2) sorting is acceptable or the array is already sorted, (3) the problem requires in-place modification with no extra data structure.

When to Reach for Two Pointers

Recognize these signals in a problem statement:

  • Find a pair or triplet in a sorted array satisfying a sum or condition
  • Remove duplicates or filter elements in-place with O(1) space
  • Maximise or minimise a value defined by two endpoints (area, distance)
  • The problem says "sorted" or sorting would not destroy useful information
  • Linked list cycle detection or finding the middle node (fast/slow variant)
