Intervals

Sort first, then scan. Interval problems reduce to one of three operations: merge overlapping ranges, insert a new range into a sorted list, or count how many ranges overlap at a point.

⏱ 5 min read 📏 Arrays 🔑 Java 8 📈 Intermediate → Advanced

Post #6 established the sort-first pattern. Interval problems are where that pattern pays off most directly. Every interval is a pair [start, end]. Sorting by start time makes a fundamental property emerge: any two intervals that overlap must be adjacent in the sorted list. A single linear scan then handles merging, gap detection, or scheduling — all in O(n) after the O(n log n) sort.

Prerequisites

Post #6 (Sorting Strategies) — sort-first pattern and custom comparators. Merge Intervals was introduced there; this post covers it again briefly and then goes deeper.

The Core Idea

Two intervals [a, b] and [c, d] overlap if and only if a <= d and c <= b. Equivalently, they do not overlap only when one ends before the other begins: b < c or d < a. Sorting by start time eliminates the second case — after sorting, a <= c is always guaranteed. The only remaining overlap check is c <= b.

Fig 1. After sorting by start, overlapping intervals are adjacent. [1,4] and [2,6] merge to [1,6]; [8,10] and [9,12] merge to [8,12].

Example 1 — Merge Intervals

LeetCode #56 · Medium · Sort then scan

Given an array of intervals, merge all overlapping intervals and return the result. This is the canonical interval problem — introduced in Post #6 and revisited here as the foundation for everything that follows.

Example

[[1,3],[2,6],[8,10],[15,18]] → [[1,6],[8,10],[15,18]]

Java 8public int[][] merge(int[][] intervals) {
    Arrays.sort(intervals, (a, b) -> a[0] - b[0]);

    List<int[]> result = new ArrayList<>();
    result.add(intervals[0]);

    for (int i = 1; i < intervals.length; i++) {
        int[] curr = intervals[i];
        int[] last = result.get(result.size() - 1);

        if (curr[0] <= last[1]) {
            last[1] = Math.max(last[1], curr[1]);  // overlap: extend end
        } else {
            result.add(curr);                       // no overlap: new group
        }
    }
    return result.toArray(new int[0][]);
}
// Time: O(n log n)  |  Space: O(n)

Example 2 — Insert Interval

LeetCode #57 · Medium · Sorted input, no sort needed

Given a list of non-overlapping intervals already sorted by start time, and a new interval, insert the new interval and merge if necessary. Return the resulting list.

Example

intervals = [[1,2],[3,5],[6,7],[8,10],[12,16]], newInterval = [4,8]
→ [[1,2],[3,10],[12,16]]

Because the input is already sorted, no sort step is needed. Process in one pass using three phases. First, add all intervals that end before the new one starts. Second, merge all intervals that overlap the new one by expanding its boundaries. Finally, add all remaining intervals.

Fig 2. Three-phase insert. Phase 1: [1,2] ends before 4 — add as-is. Phase 2: [3,5],[6,7],[8,10] all overlap [4,8] — merge to [3,10]. Phase 3: [12,16] starts after 8 — add as-is.

Java 8public int[][] insert(int[][] intervals, int[] newInterval) {
    List<int[]> result = new ArrayList<>();
    int i = 0, n = intervals.length;

    // Phase 1: add all intervals ending before newInterval starts
    while (i < n && intervals[i][1] < newInterval[0]) {
        result.add(intervals[i++]);
    }

    // Phase 2: merge all overlapping intervals into newInterval
    while (i < n && intervals[i][0] <= newInterval[1]) {
        newInterval[0] = Math.min(newInterval[0], intervals[i][0]);
        newInterval[1] = Math.max(newInterval[1], intervals[i][1]);
        i++;
    }
    result.add(newInterval);

    // Phase 3: add remaining intervals
    while (i < n) result.add(intervals[i++]);

    return result.toArray(new int[0][]);
}
// Time: O(n)  |  Space: O(n)  (input already sorted — no sort step needed)
Key Insight

Because the input is already sorted, this is O(n) — not O(n log n). The three-phase structure directly mirrors the three possible relationships between each existing interval and the new one: entirely before, overlapping, or entirely after.

When to Reach for Interval Patterns

  • The input contains pairs [start, end] representing ranges, windows, or time slots
  • "Merge overlapping" or "remove redundant" ranges — sort by start, scan and merge
  • "Insert a new interval" into an already-sorted list — three-phase scan, O(n)
  • "Minimum rooms" or "maximum overlap at any point" — sort + min-heap or sweep line
  • "Minimum removals to eliminate all overlaps" — sort by end, greedy keep earliest-ending
