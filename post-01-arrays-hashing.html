Arrays & Hashing

The universal prerequisite. Every pattern in this series relies on manipulating arrays and exploiting hash-based lookups — master these first.

⏱ 5 min read ☕ Fundamentals 🔑 Java 8 📈 Intermediate → Advanced

Arrays and hash maps are the bedrock of technical interviews. Individually they are simple structures; together they produce a transformation that appears in dozens of interview problems: a brute-force O(n²) or O(n log n) solution collapses to O(n) the moment you trade space for a hash-indexed lookup. Recognizing when and how to apply that trade is the foundational skill this post develops.

Arrays — Complexity Foundations

An array is a contiguous block of memory. Its defining property is O(1) random access — given an index, the CPU computes the address in a single instruction. Everything else flows from this.

Fig 1. Contiguous memory layout. Index-to-address calculation is constant time regardless of array size.

OperationTimeNotes
Access by indexO(1)Direct memory address calculation
Search (unsorted)O(n)Must scan every element in worst case
Search (sorted)O(log n)Binary search — covered in Post #5
Insert / delete (end)O(1)Amortised for ArrayList
Insert / delete (middle)O(n)Requires shifting all subsequent elements

Hash Maps — The O(1) Lookup Engine

A HashMap<K, V> maps keys to values in expected O(1) time for both reads and writes. Internally it maintains an array of buckets. A hash function converts the key into an integer index, determining which bucket stores the entry.

Fig 2. Hash map internals. The hash function maps any key to a bucket index in constant time, enabling O(1) average-case get and put.

Collision Handling

Two distinct keys may hash to the same bucket — a collision. Java's HashMap handles this via separate chaining: each bucket holds a linked list (or, since Java 8, a balanced tree when the chain exceeds 8 nodes). In the average case collisions are rare, preserving O(1) performance; in the pathological worst case (all keys collide) operations degrade to O(n).

OperationAverageWorst Case
put(k, v)O(1)O(n)
get(k)O(1)O(n)
containsKey(k)O(1)O(n)
remove(k)O(1)O(n)
SpaceO(n) — proportional to number of entries
Interview Context

Interviewers expect you to cite O(1) average when discussing hash maps. If probed on worst case, mention collisions and Java 8's treeification fallback — this signals genuine understanding of the underlying structure.

HashSet — Membership in O(1)

A HashSet<E> is a HashMap whose values are a dummy sentinel — it stores keys only, providing O(1) average-case contains, add, and remove. Use it when you need fast membership testing and do not require associated values.

Java 8HashSet<Integer> seen = new HashSet<>();

seen.add(7);          // O(1) — insert
seen.contains(7);     // O(1) — membership test → true
seen.remove(7);       // O(1) — delete

The Core Pattern: Trading Space for Time

The central technique of this post is straightforward: pre-process an array into a hash structure, then answer queries against that structure in O(1) rather than scanning the array repeatedly. The result is a reduction from O(n²) to O(n) at the cost of O(n) auxiliary space.

Fig 3. The two-phase pattern: one O(n) pass to build the hash structure, then O(1) lookups to answer queries. Total complexity: O(n).

Example 1 — Two Sum

LeetCode #1 · Difficulty: Easy

Given an integer array nums and a target integer target, return the indices of the two numbers that add up to target. Exactly one solution exists; the same element may not be used twice.

Example

nums = [2, 7, 11, 15], target = 9 → [0, 1] because nums[0] + nums[1] = 2 + 7 = 9

Brute Force — O(n²)

Check every pair. For each element at index i, scan every element at index j > i to see if they sum to target.

Java 8public int[] twoSumBrute(int[] nums, int target) {
    for (int i = 0; i < nums.length; i++) {
        for (int j = i + 1; j < nums.length; j++) {
            if (nums[i] + nums[j] == target) {
                return new int[] { i, j };
            }
        }
    }
    return new int[] {};   // guaranteed not reached per problem constraints
}
// Time: O(n²)  |  Space: O(1)

Optimized — O(n) with HashMap

For each element nums[i], the required complement is target - nums[i]. Store each element's index in a HashMap as you iterate. Before inserting, check whether the complement already exists in the map — if it does, the pair is found.

Fig 4. Two Sum with HashMap. At i=1, complement 2 is found in the map immediately — no inner loop needed.

Java 8public int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> map = new HashMap<>(); // value → index

    for (int i = 0; i < nums.length; i++) {
        int complement = target - nums[i];

        if (map.containsKey(complement)) {
            return new int[] { map.get(complement), i }; // found
        }

        map.put(nums[i], i); // store current element for future lookups
    }

    return new int[] {}; // unreachable per problem guarantee
}
// Time: O(n)  |  Space: O(n)

Example 2 — Group Anagrams

LeetCode #49 · Difficulty: Medium

Given an array of strings, group the anagrams together. The order of groups and order within groups does not matter.

Example

["eat","tea","tan","ate","nat","bat"] → [["eat","tea","ate"], ["tan","nat"], ["bat"]]

The key insight: two strings are anagrams if and only if their sorted character sequences are identical. Use the sorted string as a HashMap key to group all anagrams together in a single pass.

Fig 5. Sorted string as HashMap key. All anagram variants of "eat" sort to "aet" — a single key groups them automatically.

Java 8public List<List<String>> groupAnagrams(String[] strs) {
    Map<String, List<String>> map = new HashMap<>();

    for (String s : strs) {
        char[] chars = s.toCharArray();
        Arrays.sort(chars);               // sort chars to form the canonical key
        String key = new String(chars);

        // getOrDefault avoids a null check on first insertion
        map.getOrDefault(key, new ArrayList<>());
        map.putIfAbsent(key, new ArrayList<>());
        map.get(key).add(s);
    }

    return new ArrayList<>(map.values());
}
// Time: O(n · k log k)  where k = max string length (sort cost per string)
// Space: O(n · k)  — storing all characters across all strings
Optimization

Sorting each string costs O(k log k). For a fixed 26-character lowercase alphabet, you can instead build a frequency count array of length 26 as the key — reducing per-string work to O(k). The key becomes a string like "1#0#0#...#1#0" using a delimiter to prevent ambiguity.

Universal Template

The two canonical forms of the arrays-and-hashing pattern:

Java 8 — Frequency Map// Pattern A: count occurrences of each element
Map<Integer, Integer> freq = new HashMap<>();
for (int num : nums) {
    freq.put(num, freq.getOrDefault(num, 0) + 1); // increment count
}
Java 8 — Complement / Existence Map// Pattern B: check if a complement/pair exists while iterating
Map<Integer, Integer> map = new HashMap<>(); // value → index (or any metadata)
for (int i = 0; i < nums.length; i++) {
    int complement = target - nums[i];
    if (map.containsKey(complement)) {
        // found the pair — act on it
    }
    map.put(nums[i], i); // add current element after check to avoid self-pairing
}

When to Reach for Arrays & Hashing

Recognize these signals in a problem statement:

  • Find a pair, triplet, or complement that satisfies a sum or condition
  • Detect or count duplicates in an array or string
  • Group elements by a shared property (anagrams, same digit sum, etc.)
  • Check whether one array is a permutation or subset of another
  • Track the frequency of characters or numbers
